Monitoring với Prometheus & Grafana

MyMiniCloud Team

SRE & Observability Division

Giám sát sức khỏe hệ thống là yếu tố sống còn (mission-critical) trong vận hành cloud. Prometheus và Grafana là cặp bài trùng phổ biến nhất trong thế giới monitoring hiện đại, cung cấp khả năng quan sát toàn diện từ metrics đến alerting.

Prometheus – Thu thập Metrics

Prometheus hoạt động theo mô hình pull-based: nó chủ động scrape metrics từ các targets theo chu kỳ định sẵn (mặc định 15s). Các metrics được lưu trong time-series database và có thể query bằng ngôn ngữ truy vấn mạnh mẽ PromQL.

lightbulb

"Trong MyMiniCloud, chúng tôi sử dụng nginx-prometheus-exporter làm side-car pattern để chuyển đổi Nginx stub_status sang định dạng Prometheus-compatible — giải quyết triệt để lỗi parsing metrics thô."

Grafana – Trực quan hóa

Grafana kết nối đến Prometheus datasource và hiển thị dữ liệu dưới dạng dashboard với các biểu đồ trực quan: CPU usage, memory consumption, network traffic, HTTP request rates và nhiều metrics khác. Dashboard có thể được provisioning tự động qua YAML config.

query_stats

PromQL

Ngôn ngữ truy vấn mạnh mẽ cho phép tính toán, tổng hợp và lọc time-series data linh hoạt.

notifications_active

AlertManager

Hệ thống cảnh báo tự động gửi notification qua Slack, Email khi metrics vượt ngưỡng.

Alerting – Cảnh báo chủ động

Prometheus hỗ trợ cấu hình alert rules: khi một metric vượt ngưỡng (ví dụ CPU > 85%), AlertManager sẽ gửi thông báo qua Slack, Email hoặc các kênh khác. Đây là tính năng đặc biệt quan trọng trong môi trường production để phát hiện sự cố trước khi người dùng bị ảnh hưởng.

Trong MyMiniCloud, các exporter (web-1-exporter, web-2-exporter) được cấu hình như side-car containers, thu thập metrics từ Nginx stub_status và chuyển đổi sang format Prometheus-compatible. Grafana dashboard được auto-provisioning với các panels hiển thị CPU, Memory và Network traffic cho toàn bộ 9 services.
